# Dr. Vandy’s

A production e-commerce storefront for a wellness brand: product discovery, content, and purchasing.

---

## 1. Summary

Custom **Next.js** front end with **Shopify** handling catalog, cart/checkout, and payments. The site is the branded experience; transactions complete on Shopify’s infrastructure.

---

## 2. Overview

- Direct-to-consumer retail and editorial **blog** content  
- **Headless commerce**: Shopify as the commerce backend; this repo is the customer-facing app  
- Deployed and maintained for production use  

---

## 3. Highlights

- Responsive UI (mobile, tablet, desktop)  
- Product experiences backed by Shopify data  
- On-site cart UX; checkout completed via **Shopify**  
- Blog with crawlable, hierarchical URLs and standard SEO surfaces (sitemap, robots, metadata)  
- Configurable public site URL for canonical and discovery metadata  

---

## 4. Tech stack

| Area | Notes |
|------|--------|
| App | Next.js (App Router), React, JavaScript |
| Commerce & content | Shopify (Storefront API, checkout, blog) |
| Design | Figma |
| APIs | Next.js route handlers where server-side calls are needed |

---

## 5. Architecture (high level)

- **Marketing + product + blog** are rendered by this application.  
- **Shopify** provides product/catalog data and hosts checkout.  
- Environment-based configuration for Shopify and public URLs; no payment card data is collected on this origin.  

---

## 6. Local development

```bash
npm install
npm run dev
```

Configure secrets and URLs in **`.env.local`** (not committed). Use **`npm run build`** / **`npm run start`** for a production build, **`npm run lint`** for ESLint.

---

## 7. Environment variables

Set at least **Shopify Storefront** credentials and a **public site URL** for your deployment. Optional variables may apply for blog topic metafields or environment flags—refer to your team’s internal runbook or deployment config for the exact list and values.

---

## 8. Blog and SEO (overview)

- Blog paths follow a **category → topic → article** pattern for stable, shareable URLs.  
- **Sitemap** and **robots.txt** are generated by the app; point search engines at your sitemap after launch.  
- Set your production **site URL** so canonical and social metadata match the live domain.  
- After deploy: verify the property in **Google Search Console** (and optionally other webmaster tools) and submit the sitemap.  

Implementation details are intentionally omitted here.
